## 1. Functional Requirements
- FR1: System shall send emergency alerts without internet
- FR2: System shall detect abnormal environmental conditions
- FR3: System shall support SOS/manual trigger
- FR4: System shall transmit location data
- FR5: System shall notify command center

## 2. Non-Functional Requirements
- NFR1: Low power consumption
- NFR2: High reliability during disasters
- NFR3: Scalable architecture
- NFR4: Secure communication
- NFR5: Low latency alerts

## 3. Hardware Requirements
- Microcontroller (ESP32 / STM32 / Arduino-class)
- LoRa / BLE / RF module
- GPS module (optional)
- Battery / Power management

## 4. Software Requirements
- Embedded firmware
- Gateway control software
- Dashboard / Monitoring UI

## 5. Communication Requirements
- LoRa / LoRaWAN / Mesh protocol

## 6. Constraints
- Must operate during power/network failure
- Must remain cost-effective

## 7. Success Criteria
- Alerts delivered within defined latency
- Stable operation under adverse conditions
